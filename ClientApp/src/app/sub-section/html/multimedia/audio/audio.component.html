HTML-элемент <audio> используется для встраивания звукового содержимого в документы. Он может содержать один или несколько аудиоисточников, представленных с помощью атрибута src или элемента <source>: браузер выберет наиболее подходящий. Он также может быть местом назначения для потокового мультимедиа с использованием MediaStream.

Интерактивный пример

В приведенном выше примере показано простое использование элемента <audio>. Подобно элементу <img>, мы включаем путь к медиафайлу, который хотим отобразить, внутри атрибута src, мы можем включить другие атрибуты, чтобы указать такую информацию, как ширина и высота видео, хотим ли мы, чтобы оно автоматически воспроизводилось и зацикливалось, хотим ли мы отображать элементы управления видео по умолчанию в браузере и т. д.

Контент внутри открывающего и закрывающего тегов <audio> </audio> отображается как запасной вариант в браузерах, не поддерживающих этот элемент.

Атрибуты

autoplay
Логический атрибут, если указано, видео автоматически начинает воспроизводиться, как только оно может это сделать, не останавливаясь для завершения загрузки данных. Чтобы удалить автовоспроизведение, атрибут нужно удалить вообще.
controls
Если этот атрибут присутствует, браузер предложит элементы управления, позволяющие пользователю управлять воспроизведением видео, включая громкость, поиск и паузу/возобновление воспроизведения.
loop
Логический атрибут, если указано, браузер будет автоматически возвращаться к началу при достижении конца видео.
muted
Логический атрибут, указывающий настройку по умолчанию для звука, содержащегося в видео. Если установлено, звук изначально будет отключен.
preload
Этот перечисляемый атрибут предназначен для того, чтобы предоставить браузеру подсказку о том, что, по мнению автора, приведет к наилучшему взаимодействию с пользователем в отношении того, какой контент загружается перед воспроизведением видео.
src
URL-адрес видео для встраивания. Это необязательно, вместо этого вы можете использовать элемент <source> в блоке видео, чтобы указать видео для встраивания.

Рекомендации по использованию

Не все браузеры поддерживают одни и те же типы файлов и аудиокодеки, вы можете указать несколько источников внутри вложенных элементов <source>, и тогда браузер будет использовать первый из них, который он понимает.

Другие примечания по использованию:

  • Если вы не укажете атрибут controls, аудиоплеер не будет включать в себя элементы управления браузера по умолчанию, вы можете создавать собственные настраиваемые элементы управления с помощью JavaScript и API HTMLMediaElement.
  • Чтобы обеспечить точный контроль над вашим аудио содержимым, HTMLMediaElements запускает множество различных событий. Э Это также дает возможность отслеживать процесс извлечения аудио, чтобы вы могли следить за ошибками или определять, когда доступно достаточно, чтобы начать воспроизведение или манипулировать им.
  • Вы также можете использовать API веб-аудио для прямого создания аудиопотоков и управления ими из кода JavaScript вместо потоковой передачи уже существующих аудиофайлов.
  • Элементы <audio> не могут иметь субтитры или титры, связанные с ними так же, как элементы <video>.

Обнаружение добавления и удаления треков

Вы можете определить, когда дорожки добавляются и удаляются из элемента <audio>, используя события addtrack и removetrack. Однако эти события не отправляются непосредственно в сам элемент <audio>. Вместо этого они отправляются в объект списка дорожек в HTMLMediaElement элемента <audio>, который соответствует типу дорожки, добавленной к элементу.

Проблемы доступности

Аудио с разговорным диалогом должно содержать как подписи, так и стенограммы, которые точно описывают его содержание. Субтитры, указанные с помощью WebVTT, позволяют людям с потерей слуха понимать содержание аудиозаписи во время ее воспроизведения, а стенограммы позволяют людям, которым требуется дополнительное время, просматривать содержимое записи в темпе и формате, им удобно.

Если используются службы автоматического создания субтитров, важно проверить сгенерированный контент, чтобы убедиться, что он точно представляет исходный звук.

Элемент <audio> не поддерживает WebVTT напрямую. Вам нужно будет найти библиотеку или фреймворк, которые предоставляют вам такую возможность, или написать код для отображения подписей самостоятельно. Один из вариантов — воспроизвести аудио с помощью элемента <video>, который поддерживает WebVTT.

В дополнение к разговорному диалогу субтитры и стенограммы также должны идентифицировать музыку и звуковые эффекты, которые передают важную информацию. Это включает в себя эмоции и тон.
