Элементы формы, связанные по смыслу, можно сгруппировать между тегами <fieldset> и </fieldset>. Это бывает особенно полезно при создании длинных форм. Большинство браузеров отобразят <fieldset> в виде рамки вокруг группы элементов формы.

Элемент <legend>, содержащий надпись для обозначения группы элементов формы, как правило, помещают сразу после открывающего тега элемента <fieldset>.

Интерактивный пример

Как показано в приведенном выше примере, элемент <fieldset> обеспечивает группировку части HTML-формы, а вложенный элемент <legend> обеспечивает заголовок для <fieldset>. Требуется несколько атрибутов, наиболее заметным из которых является form, которая может содержать идентификатор <form> на той же странице, что позволяет вам сделать <fieldset> частью этой <form>, даже если он не вложен в нее, и disabled, что позволяет отключить <fieldset> и все его содержимое за один раз.

Атрибуты

disabled
Если этот логический атрибут установлен, все элементы управления формы, которые являются потомками <fieldset>, отключаются, то есть они недоступны для редактирования и не будут отправлены вместе с <form>. Они не будут получать никаких событий просмотра, таких как щелчки мышью или события, связанные с фокусом. По умолчанию браузеры отображают такие элементы управления серым цветом. Обратите внимание, что элементы формы внутри элемента <legend> не будут отключены.
form
Этот атрибут принимает значение атрибута id элемента <form>, частью которого должен быть <fieldset>, даже если он не находится внутри формы. Обратите внимание, что использование этого сбивает с толку — если вы хотите, чтобы элементы <input> внутри <fieldset> были связаны с формой, вам нужно использовать атрибут формы непосредственно для этих элементов.
name
Имя, связанное с группой.
