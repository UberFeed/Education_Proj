Все элементы формы помещаются в тег <form>. Для него всегда должен быть указан атрибут action, кроме того, для этого элемента часто устанавливают атрибуты method и id.

Элемент <input> используется для создания компонентов формы нескольких типов. Значение атрибута type указывает этот тип.

У каждого элемента формы должна быть своя надпись, или метка, создаваемая при помощи элемента <label> и упрощающая пользование сайтом людям с нарушениями зрения.

Интерактивный пример

Элемент <lable> можно использовать двумя способами. Окружить им и метку и сам элемент формы, как показано справа на примере поля ввода текста. Указать отдельно от элемента формы, но при этом использовать атрибут for для соотнесения с нужным элементом, как показано на примере переключателей.

Значение атрибута for - это значение атрибута id требуемого элемента формы. Данный прием присвоения меток может быть использован со всеми элементами формы.

Значения атрибута type

То, как работает <input>, значительно различается в зависимости от значения его атрибута type, поэтому различные значения будут рассмотрены более детально.

button
Кнопка без поведения по умолчанию, отображающая значение атрибута value
checkbox
Флажок, позволяющий выбирать/отменять выбор отдельных значений. С помощью флажков посетители сайта могут выбирать из нескольких вариантов ответа, а также отменять выбор.
color
Элемент управления для указания цвета; открытие палитры цветов, когда она активна в поддерживаемых браузерах.
date
Элемент управления для ввода даты (год, месяц и день, без времени). Открывает средство выбора даты или числовые колеса для года, месяца, дня, когда они активны в поддерживающих браузерах.
datetime-local
Элемент управления для ввода даты и времени без часового пояса. Открывает средство выбора даты или числовые колеса для компонентов даты и времени, когда они активны в поддерживающих браузерах.
email
Поле для редактирования адреса электронной почты. Выглядит как ввод текста, но имеет параметры проверки и соответствующую клавиатуру для поддержки браузеров и устройств с динамической клавиатурой.
file
Элемент управления, позволяющий пользователю выбрать файл. Используйте атрибут accept, чтобы определить типы файлов, которые может выбирать элемент управления.
hidden
Элемент управления, который не отображается, но значение которого отправляется на сервер.
image
Если вы хотите использовать собственное изображение для кнопки подтверждения, то вам потребуется присвоить атрибуту type значение image. При этом атрибуты src, width, height и alt будут работать точно также, как и для элемента <img>
month
Элемент управления для ввода месяца и года без часового пояса.
number
Элемент управления для ввода числа. Отображает счетчик и добавляет проверку по умолчанию. Отображает цифровую клавиатуру на некоторых устройствах с динамической клавиатурой.
password
Однострочное текстовое поле, значение которого скрыто.
radio
Радиокнопка, позволяющая выбрать одно значение из нескольких вариантов с одинаковым значением атрибута name.
range
Элемент управления для ввода числа, точное значение которого не важно. Отображается как виджет диапазона со средним значением по умолчанию. Используется вместе с min и max для определения диапазона допустимых значений.
search
Однострочное текстовое поле для ввода строк поиска. Разрывы строк автоматически удаляются из входного значения. Может включать значок удаления в поддерживаемых браузерах, который можно использовать для очистки поля. Отображает значок поиска вместо клавиши ввода на некоторых устройствах с динамической клавиатурой.
tel
Элемент управления для ввода номера телефона. Отображает клавиатуру телефона на некоторых устройствах с динамической клавиатурой.
text
Значение по умолчанию. Однострочное текстовое поле. Разрывы строк автоматически удаляются из входного значения.
time
Элемент управления для ввода значения времени без часового пояса.
url
Поле для ввода URL. Выглядит как ввод текста, но имеет параметры проверки и соответствующую клавиатуру для поддержки браузеров и устройств с динамической клавиатурой.
week
Поле для ввода URL. Выглядит как ввод текста, но имеет параметры проверки и соответствующую клавиатуру для поддержки браузеров и устройств с динамической клавиатурой.

Атрибуты

Элемент input такой мощный благодаря своим атрибутам. Атрибут type, описанный с примерами выше, является наиболее важным. Поскольку каждый элемент input, независимо от типа, основан на интерфейсе HTMLInputElement, технически они имеют один и тот же набор атрибутов. Однако на самом деле большинство атрибутов влияют только на определенное подмножество типов ввода. Кроме того, то, как некоторые атрибуты влияют на ввод, зависит от типа ввода, по-разному влияя на разные типы ввода.

В этом разделе представлена таблица со списком всех атрибутов с кратким описанием. За этой таблицей следует список, описывающий каждый атрибут более подробно, а также типы входных данных, с которыми они связаны. Те из них, которые являются общими для большинства или всех типов ввода, более подробно определены ниже. Атрибуты, уникальные для конкретных типов ввода, или атрибуты, общие для всех типов ввода, но имеющие особое поведение при использовании для данного типа ввода, вместо этого документируются на страницах этих типов.

Примеры

Автозаполнение элемента управления.

Указывает, может ли значение элементов <input> автоматически заполняться браузером.

autocomplete='off' — пользователь должен явно вводить значение в это поле для каждого использования.

autocomplete='on' — браузер может автоматически дополнять значение на основе значений, которые пользователь ввел во время предыдущих использований.

Атрибут автозаполнения принимает в качестве своего значения строку, разделенную пробелами, которая описывает, какие функции автозаполнения должны предоставляться входными данными, если таковые имеются. Типичная реализация автозаполнения вызывает предыдущие значения, введенные в то же поле ввода, но могут существовать и более сложные формы автозаполнения. Например, браузер может интегрироваться со списком контактов устройства для автозаполнения адресов электронной почты в поле ввода электронной почты.

Использование шаблонов для валидации формы.

Возможно, вы встречали во Всемирной паутине формы, выводящие предупреждения в случае, если не все поля были правильно заполнены. Эта реакция называется валидацией данных формы. Традиционно валидация введенных данных осуществляется с помощью специальных сценариев JavaScript.

В языке HTML5 появилась собственная технология валидации введенных данных: теперь эта работа перекладывается на веб-браузер. Валидация позволяет гарантировать, что пользователь введет данные в формате, с которым сможет работать сервер.
