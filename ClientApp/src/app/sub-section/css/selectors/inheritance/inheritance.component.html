В CSS наследование - это механизм, позволяющий элементам наследовать значения свойств от своих родительских элементов. Таким образом, если свойство не определено для элемента, оно может быть унаследовано от его родительского элемента.

Каскад в CSS описывает порядок применения стилей к элементам HTML. Когда браузер обрабатывает HTML-документ и находит соответствующий элемент, он начинает применять стили к этому элементу, используя различные источники стилей, такие как внутренние стили, внешние стили и стили по умолчанию.

Наследование свойств

Спецификацией CSS предусмотрено наследование свойств, относящихся к текстовому содержимому страницы, таких как color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space и word-spacing. Во многих случаях это удобно, так как не нужно задавать размер шрифта и семейство шрифтов для каждого элемента веб-страницы.

Свойства, относящиеся к форматированию блоков, не наследуются. Это background, border, display, float и clear, height и width, margin, min/max-height и min/max-width, outline, overflow, padding, position, text-decoration, vertical-align и z-index.

Принудительное наследование

В CSS принудительное наследование может быть задано с помощью ключевого слова inherit, которое указывает, что элемент должен наследовать значение свойства от своего родительского элемента. Если значение свойства не было задано для родительского элемента, браузер перейдет к следующему родительскому элементу, пока не найдет значение свойства.

Например, если вы хотите, чтобы дочерний элемент наследовал цвет текста от своего родительского элемента, как в примере выше.

Каскад

Каскадность в CSS означает, что стили применяются к элементам в зависимости от их приоритетности и специфичности, а также от порядка следования правил в таблице стилей.

CSS-правила могут быть определены на разных уровнях: внутри элемента (inline), внутри блока (internal) и в отдельном файле (external). Каждое правило имеет вес, который определяет его приоритетность при каскадировании. Вес правила определяется на основе типа селектора, количества селекторов, наличия псевдоклассов и псевдоэлементов.

Если к одному и тому же элементу применяется два или более правил, важно понимать, которое из них получит приоритет.

Последнее правило
При наличии двух одинаковых селекторов приоритет получает последний. В примере вы можете видеть, как второй селектор i получает приоритет над первым.
Специфичность
Если один селектор является более специфичным, чем остальные, то он получает приоритет. В данном примере:
  • h1 является более специфичным, чем *;
  • р Ь является более специфичным, чем р;
  • p#intro является более специфичным, чем р.

Для каждого правила браузер вычисляет специфичность селектора, и если у элемента имеются конфликтующие объявления свойств, во внимание принимается правило, имеющее наибольшую специфичность. Значение специфичности состоит из четырех частей: 0, 0, 0, 0. Специфичность селектора определяется следующим образом:

  • для #id добавляется 0, 1, 0, 0;
  • для .class добавляется 0, 0, 1, 0;
  • для каждого элемента и псевдоэлемента добавляется 0, 0, 0, 1;
  • для встроенного стиля, добавленного непосредственно к элементу — 1, 0, 0, 0;
  • универсальный селектор не имеет специфичности.

В результате к элементу применятся те правила, специфичность которых больше.

Важность
Вы можете добавить ключевое слово !important после значения любого свойства, чтобы указать, что его следует считать самым важным из правил, относящихся к одному и тому же элементу.
